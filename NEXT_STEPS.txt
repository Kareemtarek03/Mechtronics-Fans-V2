Create repo on GitHub, then:
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git branch -M main
git push -u origin main

STEP 4: DEPLOY ON RENDER.COM
----------------------------
1. Go to https://render.com
2. Sign up/Login with GitHub
3. Click "New +" → "Web Service"
4. Connect your GitHub repository
5. Configure:
   - Name: mecha-eg-fullstack
   - Build Command: npm run render-build
   - Start Command: npm start
6. Click "Create Web Service"
7. Wait for deployment (5-10 minutes)
8. Access your app at the provided URL!

================================================================================
RENDER.COM BUILD PROCESS:
================================================================================

When you deploy, Render will:
1. Run: npm install (backend dependencies)
2. Run: cd client && npm install (frontend dependencies)
3. Run: cd client && npm run build (build React app)
4. Run: npm start (start Express server)

The Express server will:
- Serve API endpoints at /api/fan-data/*
- Serve React app for all other routes
- Handle both frontend and backend in one service

TROUBLESHOOTING:
================================================================================

Issue: Copy script doesn't work
Solution: Manually copy files as described in QUICK_START.md

Issue: Build fails on Render
Solution: Check logs, ensure all files are committed to Git

Issue: App works locally but not on Render
Solution: Check Render logs, verify file paths are relative

Issue: "Cannot find module" errors
Solution: Ensure output.json and MotorData.json are in server/ folder
